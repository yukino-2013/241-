 智能运维助手测试用例
 
测试范围：后端工具函数（CPU、内存、服务检测）、智能体对话接口、前端交互。
测试环境：Linux 环境（主要），需确保 `systemctl` 可用；非 Linux 环境可使用备用进程检测函数。
依赖服务：智谱AI API、后端服务（`http://localhost:8000`）、前端服务（`http://localhost:8501`）。

---

 测试场景

 一、正常场景

| 编号 | 测试项         | 输入/操作                                         | 期望输出                                                                 |
|------|----------------|---------------------------------------------------|--------------------------------------------------------------------------|
| TC01 | CPU 查询       | 用户输入：“CPU使用率是多少？”                      | 返回类似“当前CPU使用率为XX%”的信息，XX 为 0-100 之间的数值。             |
| TC02 | 内存查询       | 用户输入：“内存使用情况”                          | 返回类似“内存已使用X.XGB/共X.XGB，使用率XX%”的信息。                    |
| TC03 | 服务检查（存在） | 用户输入：“检查 nginx 服务”                       | 返回“服务nginx正在运行” （若 nginx 实际运行）或“服务nginx未运行”。      |

 二、边界场景

| 编号 | 测试项         | 输入/操作                                         | 期望输出                                                                 |
|------|----------------|---------------------------------------------------|--------------------------------------------------------------------------|
| TC04 | CPU 空闲       | 在系统空闲时查询 CPU                              | 返回的 CPU 使用率应接近 0%（允许 0~5% 波动）。                           |
| TC05 | 内存满载       | 模拟内存使用率达到 90% 以上时查询                  | 返回的内存使用率应与实际一致，误差不超过 5%。                             |
| TC06 | 服务名大小写   | 用户输入：“检查 Nginx 服务”（N 大写）            | 应正确识别并返回与 TC03 一致的结果（忽略大小写）。                        |

 三、异常场景

| 编号 | 测试项         | 输入/操作                                         | 期望输出                                                                 |
|------|----------------|---------------------------------------------------|--------------------------------------------------------------------------|
| TC07 | 不存在的服务   | 用户输入：“检查 abcdefg 服务”                     | 返回“服务abcdefg未运行”。                                                |
| TC08 | 后端服务未启动 | 前端启动但后端未运行，用户发送问题                 | 前端显示友好的连接错误提示：“无法连接到后端服务。请确认后端已启动...”。   |
| TC09 | API 超时       | 模拟后端响应延迟超过 15 秒                         | 前端显示超时提示：“请求超时，后端响应过慢，请稍后重试。”                  |
| TC10 | 无效问题       | 用户输入与系统状态无关的问题，如“今天天气怎么样？” | 智能体应给出合理的兜底回复（如“抱歉，我只能查询服务器状态”或类似提示）。   |
| TC11 | 缺少 API Key   | 启动后端前未配置 `ZHIPU_API_KEY`                   | 后端启动报错或调用大模型时返回明确错误信息，前端应展示相应错误。          |
