智能运维助手 - 小组协作方案（最终版）
一、项目概述
题目：智能运维助手设计与实现
一句话介绍：通过自然语言对话查询服务器状态（CPU、内存、服务运行情况）的AI智能体。
核心功能（MVP）：
查询CPU使用率
查询内存使用情况
检查指定服务（如nginx）是否运行
技术栈：
后端：Python + LangChain +某AI API + FastAPI
前端：Streamlit
协作：Git + GitHub
二、三人分工（最终版）
说明：C也参与开发（分担部分代码），但测试和文档仍由其主责。
三、协作方式
1. 代码管理（Git）
仓库地址：https://github.com/用户名/smart-ops-assistant（C创建）
分支策略：
main：稳定版本（演示用）
dev：日常开发，所有人向此分支推送
每天结束前，每人将当天修改推送到 dev，C负责合并
2. 沟通机制
每日站会：晚上9点微信群文字汇报（今天做了什么、明天计划、有无困难）
每周同步：周末语音15分钟，检查进度
即时沟通：遇到问题随时在群里问，超过30分钟未解决@所有人
3. 文档与测试
所有文档放在 docs/ 文件夹，用Markdown编写
测试用例由C维护，每周执行一次，记录结果
四、第一周详细任务（按角色）
A角色（后端核心）
目标：完成环境搭建、工具函数、智能体封装、API接口。
第一周交付：代码已推送至仓库，后端服务可本地运行。
B角色（前端界面）
目标：搭建Streamlit环境，完成聊天界面，能调用后端API。
第一周交付：app.py代码已推送，界面可运行（暂时用假数据或等待后端）。
C角色（辅助开发 + 测试 + 文档）
目标：建立Git仓库，协助A和B，编写测试用例和文档。
